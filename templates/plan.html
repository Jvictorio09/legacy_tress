1. Project Overview & Goals

LegacyTress is a luxury braiding and natural hair care salon located in Margate, FL. This document outlines the complete technical plan for designing, building, and deploying a high-converting, AI-powered website that positions LegacyTress as the premium protective styling destination in South Florida.

Primary Objectives

  • Convert website visitors into booked appointments through seamless UX and AI-guided service selection
  • Educate clients on services, pricing, hair care, and scalp wellness with rich, expert content
  • Enable virtual hairstyle try-on before booking to reduce hesitation and increase confidence
  • Collect and nurture leads with GoHighLevel CRM automation and AI follow-up sequences
  • Offer flexible Afterpay / payment plan options via Square for every service tier
  • Power AI consultation, follow-up, aftercare, and rebooking sequences automatically
  • Rank on Google for luxury braiding searches across Broward County and South Florida
  • Reflect the brand's chic, elegant, wellness-focused aesthetic at every client touchpoint

Brand Aesthetic Direction

The website must feel like a luxury braid spa, not a typical hair salon listing. Design direction: chic, elegant, soft, feminine, professional, and wellness-focused. Clients should feel that booking LegacyTress is a premium self-care experience.

2. Architecture Overview

Rather than building the entire site inside GoHighLevel's page builder, this architecture gives LegacyTress full ownership of the frontend and backend — with GoHighLevel retained as the CRM, automation, and AI chatbot engine. This delivers superior performance, design flexibility, and long-term scalability at a low monthly infrastructure cost.

6. Virtual Try-On Integration

Phase 1 (MVP) — Link-Out Approach

To get to market quickly, the try-on page guides clients to an existing AI hairstyle tool (e.g., YouCam Makeup web app) with LegacyTress-branded context, style references, and colour guides. The client is then directed back to book the matching service. This unblocks the launch immediately while v2 is developed.

Phase 2 — Replicate.com API Integration

A hairstyle transfer model on Replicate.com accepts a selfie and a style reference image, returning an AI-composited preview. Django handles the API call, stores the result in Cloudflare R2, and returns the preview URL to the frontend. Affordable, fully API-driven, no third-party licensing required.

# Django view (simplified)
import replicate

def generate_preview(selfie_url, style_ref_url):
    output = replicate.run(
        'model/hairstyle-transfer:version',
        input={'image': selfie_url, 'style_image': style_ref_url}
    )
    return output[0]  # URL of generated image

Phase 3 — Perfect Corp API (Premium)

Perfect Corp's enterprise YouCam API delivers photorealistic results with a large style library. Recommended as the long-term solution once the website is generating consistent revenue. Requires a licensing agreement with Perfect Corp.

7. Payment Integration — Square + Afterpay

Phase 1 — Square Payment Links (Recommended Launch Path)

Square Payment Links are the fastest path to accepting Afterpay on the website. The Django backend generates a payment link on demand via the Square Python SDK. Square's hosted checkout natively supports Afterpay/Clearpay, Apple Pay, Google Pay, and credit/debit cards.

from square.client import Client
import uuid

def create_deposit_link(service_name, amount_cents, redirect_url):
    result = client.checkout.create_payment_link({
        'idempotency_key': str(uuid.uuid4()),
        'quick_pay': {
            'name': service_name,
            'price_money': {'amount': amount_cents, 'currency': 'USD'},
            'location_id': settings.SQUARE_LOCATION_ID,
        },
        'checkout_options': {
            'redirect_url': redirect_url,
            'accepted_payment_methods': {'afterpay_clearpay': True}
        }
    })
    return result.body['payment_link']['url']

8. GoHighLevel CRM Integration

GoHighLevel is retained as the CRM and automation engine. The Django backend acts as the data bridge — every form submission, consultation, try-on request, and booking event creates or updates a GHL contact via the REST API, with the correct tags, pipeline stage, and custom fields applied automatically.

Django GHL Client

import httpx
from django.conf import settings

def push_contact_to_ghl(data: dict):
    headers = {'Authorization': f'Bearer {settings.GHL_API_KEY}'}
    payload = {
        'firstName': data['first_name'],
        'email':     data['email'],
        'phone':     data['phone'],
        'tags':      data.get('tags', []),
        'customField': {
            'hair_texture':    data.get('hair_texture'),
            'desired_style':   data.get('desired_style'),
            'scalp_condition': data.get('scalp_condition'),
        }
    }
    resp = httpx.post('https://rest.gohighlevel.com/v1/contacts/',
                      json=payload, headers=headers)
    return resp.json()

16. Security & Compliance

  • All API keys stored as Railway environment variables — never in source code or version control
  • Django CSRF protection enabled on all form endpoints
  • DRF API rate-limiting via django-ratelimit to prevent abuse and scraping
  • Cloudflare WAF (Web Application Firewall) active on all traffic
  • Square PCI-compliant checkout — card data never touches the Django server
  • GDPR-friendly: cookie consent banner, privacy policy page, data deletion on request
  • File uploads validated and type-checked before storage in Cloudflare R2
  • HTTPS enforced everywhere via Cloudflare Full Strict SSL
  • Django security middleware: HSTS, X-Frame-Options, Content Security Policy headers
  • GHL API key scoped to minimum required permissions (contacts + workflows only)
  • Automated PostgreSQL backups on Railway — daily, 7-day retention
  • Uptime monitoring via Railway alerts or Better Uptime
